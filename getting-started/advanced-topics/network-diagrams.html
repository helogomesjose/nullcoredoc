🕸️ Diagramas de rede

Aqui, fornecemos diagramas claros e estruturados para ajudá -lo a entender como vários componentes da rede interagem em diferentes configurações. Esta documentação foi projetada para ajudar os usuários MacOS/Windows e Linux. Cada cenário é ilustrado usando diagramas de sereia para mostrar como as interações são configuradas, dependendo das diferentes configurações do sistema e estratégias de implantação.

Mac OS/Windows Setup Options 🖥️

Ollama no anfitrião, Open Webui em contêiner

Neste cenário,Ollamacorre diretamente na máquina host enquantoOpen WebUIopera dentro de um contêiner do docker.

Ollama e Open Webui em Compose Stack

AmbosOllamaeOpen WebUIsão configurados na mesma pilha de compõe do Docker, simplificando as comunicações de rede.

Ollama e webui aberto, redes separadas

Aqui,OllamaeOpen WebUIsão implantados em redes de docker separadas, potencialmente levando a problemas de conectividade.

Abrir webui na rede host

Nesta configuração,Open WebUIUtiliza a rede host, que afeta sua capacidade de se conectar em determinados ambientes.

Opções de configuração do Linux 🐧

Ollama no anfitrião, Open Webui em contêiner (Linux)

Este diagrama é específico para a plataforma Linux, comOllamacorrendo no host eOpen WebUIimplantado dentro de um contêiner do docker.

Ollama e Open Webui em Compose Stack (Linux)

Uma configuração onde ambosOllamaeOpen WebUIresidir na mesma pilha de compor do docker, permitindo uma rede direta no Linux.

Ollama e Open Webui, redes separadas (Linux)

Um cenário em queOllamaeOpen WebUIestão em diferentes redes de docker em um ambiente Linux, que pode dificultar a conectividade.

Open Webui na Rede Host, Ollama no Host (Linux)

Um layout ideal onde ambosOpen WebUIeOllamaUse a rede do host, facilitando a interação perfeita nos sistemas Linux.

Cada configuração aborda diferentes estratégias de implantação e configurações de rede para ajudá -lo a escolher o melhor layout para seus requisitos.
