Parâmetros de bate -papo

Dentro de webui aberto, existem três níveis para definir umPrompt de sistemaeParâmetros avançados: Base por bate-papo, base por modelo e base por conta. Esse sistema hierárquico permite flexibilidade, mantendo a administração e o controle estruturados.

Sistema Prompt e Parâmetros Avançados Cabe

NívelDefiniçãoPermissões de modificaçãoSubstituir recursos
Por bate-papoParâmetros de prompt de sistema e avançado para uma instância de bate -papo específicaOs usuários podem modificar, mas não podem substituir as configurações específicas do modeloRestringido de substituir configurações específicas do modelo
Por contaParâmetros de prompt de sistema padrão e avançado para uma conta de usuário específicaOs usuários podem definir, mas podem ser substituídos por configurações específicas do modeloAs configurações do usuário podem ser substituídas por configurações específicas do modelo
Por modeloParâmetros de prompt de sistema padrão e avançado para um modelo específicoOs administradores podem definir, os usuários não podem modificarAs configurações específicas do administrador têm precedência, as configurações do usuário podem ser substituídas

1.Base por bate-papo:

  • Descrição: A configuração de base por bate-papo refere-se ao prompt do sistema e parâmetros avançados configurados para uma instância de bate-papo específica. Essas configurações são aplicáveis ​​apenas à conversa atual e não afetam os bate -papos futuros.
  • Como definir: Os usuários podem modificar o prompt do sistema e parâmetros avançados para uma instância de bate-papo específica na barra lateral da direitaControles de bate -papoSeção em Webui aberto.
  • Substituir recursos: Os usuários estão restritos de substituir oPrompt de sistemaou específicoParâmetros avançadosjá definido por um administrador por modelo (#2). Isso garante consistência e adesão às configurações específicas do modelo.
Exemplo de uso de uso
Base por bate-papo

Suponha que um usuário deseje definir um prompt de sistema personalizado para uma conversa específica. Eles podem fazer isso acessando oControles de bate -paposeção e modificação doPrompt de sistemacampo. Essas alterações se aplicarão apenas à sessão de bate -papo atual.

2.Base por conta:

  • Descrição: A configuração de base por conta refere-se ao prompt do sistema padrão e aos parâmetros avançados configurados para uma conta de usuário específica. Quaisquer alterações específicas do usuário podem servir como fallback em situações em que as configurações de nível inferior não são definidas.
  • Como definir:Em geralseção doConfiguraçõesMenu em Webui aberto.
  • Substituir recursos: Os usuários têm a capacidade de definir seu próprio sistema de sistema em sua conta, mas devem estar cientes de que esses parâmetros ainda podem ser substituídos se um administrador já tiver definido oPrompt de sistemaou específicoParâmetros avançadosem uma base por modelo para o modelo específico que está sendo usado.
Exemplo de uso de uso
Base por conta

Suponha que um usuário queira definir seu próprio sistema de sistema para sua conta. Eles podem fazer isso acessando oConfiguraçõesmenu e modificando oPrompt de sistemacampo.

3.Base por modelo:

  • Descrição: A configuração de base por modelo refere-se ao prompt do sistema padrão e parâmetros avançados configurados para um modelo específico. Essas configurações são aplicáveis ​​a todas as instâncias de bate -papo usando esse modelo.
  • Como definir: Os administradores podem definir o prompt do sistema padrão e parâmetros avançados para um modelo específico dentro doModelosseção doEspaço de trabalhoem webui aberto.
  • Substituir recursosUsuárioas contas estão restritos de modificar oPrompt de sistemaou específicoParâmetros avançadosem uma base por modelo (#3). Essa restrição impede que os usuários alterem inadequadamente as configurações padrão.
  • Preservação do comprimento do contexto:Quando um modeloPrompt de sistemaou específicoParâmetros avançadossão definidos manualmente noEspaço de trabalhoseção por um administrador, dissePrompt de sistemaou definido manualmenteParâmetros avançadosnão pode ser substituído ou ajustado por conta de conta dentro doEm geralconfigurações ouControles de bate -paposeção por aUsuárioconta. Isso garante a consistência e evita a recarga excessiva do modelo sempre que a configuração do comprimento do contexto de um usuário muda.
  • Precedência do modelo:Se um modeloPrompt de sistemaou específicoParâmetros avançadosO valor é predefinido na seção de espaço de trabalho por um administrador, qualquer comprimento de contexto alterações feitas por umUsuárioconta noEm geralconfigurações ouControles de bate -papoA seção será desconsiderada, mantendo o valor pré-configurado para esse modelo. Estar avisado de que os parâmetros deixados intocados por umAdminconta ainda pode ser ajustada manualmente por umUsuárioconta por conta ou base por bate-papo.
Exemplo de uso de uso
Base por modelo

Suponha que um administrador deseje definir um prompt de sistema padrão para um modelo específico. Eles podem fazer isso acessando oModelosseção e modificação doPrompt de sistemacampo para o modelo correspondente. Quaisquer instâncias de bate -papo usando este modelo usarão automaticamente os parâmetros prompts e avançados do sistema do modelo.

Otimize as configurações de prompt do sistema para máxima flexibilidade

Dicas de bônus

Esta dica se aplica a administradores e contas de usuário. Para alcançar a máxima flexibilidade com os avisos do seu sistema, recomendamos considerar a seguinte configuração:

  • Atribua seu prompt do sistema primário (ou seja, para dar a um LLM um personagem definidor) você quer usar em seuEm geralconfiguraçõesPrompt de sistemacampo. Isso o define em um nível por conta e permite que ele funcione como o sistema solicite todos os seus LLMs sem exigir ajustes dentro de um modelo doEspaço de trabalhoseção.

  • Para o seu prompt de sistema secundário (ou seja, para dar a um LLM uma tarefa para executar), escolha se deve colocá -lo noPrompt de sistemacampo dentro doControles de bate -papobarra lateral (por bate-papo) ou oModelosseção doEspaço de trabalhoSeção (por modelo) para os administradores, permitindo que você os defina diretamente. Isso permite que o seu sistema de sistema no nível da conta funcione em conjunto com o sistema de sistema de nível por bate-papo fornecido porControles de bate -papo, ou o sistema de sistema de nível por modelo fornecido porModelos

  • Como administrador, você deve atribuir seus parâmetros LLM em base por modelo usandoModelosSeção para flexibilidade ideal. Para ambos os avisos do sistema secundário, certifique-se de defini-los de uma maneira que maximize a flexibilidade e minimize os ajustes necessários em diferentes instâncias por conta ou por bate-papo. É essencial para sua conta de administrador e todas as contas de usuário para entender a ordem prioritária por que sistema solicitaControles de bate -papoe oModelosa seção será aplicada aoLlm
