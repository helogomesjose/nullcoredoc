📚 Slots

OPromptsseção doWorkspaceNa Webui Open, permite que os usuários criem, gerenciem e compartilhem prompts personalizados. Esse recurso simplifica as interações com os modelos de IA, permitindo que os usuários salvem os prompts usados ​​com frequência e acessá -los facilmente através dos comandos Slash.

Gerenciamento imediato

A interface de prompts fornece vários recursos importantes para gerenciar seus avisos personalizados:

  • Criar: Projete novos avisos com títulos personalizáveis, níveis de acesso e conteúdo.
  • Compartilhar: Compartilhar instruções com outros usuários com base nas permissões de acesso configuradas.
  • Controle de acesso: Defina permissões de visibilidade e uso para cada prompt (consultePermissõesPara mais detalhes).
  • Comandos de barra: Acesse os avisos de acesso rapidamente usando comandos de barra personalizados durante as sessões de bate -papo.

Criando e editando prompts

Ao criar ou editar um prompt, você pode definir as seguintes configurações:

  • Título: Dê ao seu prompt um nome descritivo para facilitar a identificação.
  • Acesso: Defina o nível de acesso para controlar quem pode visualizar e usar o prompt.
  • Comando: Defina um comando Slash que acionará o prompt (por exemplo,,/summarize
  • Conteúdo rápido: Escreva o texto rápido real que será enviado ao modelo.

Variáveis ​​rápidas

O Open Webui suporta dois tipos de variáveis ​​para tornar seus avisos mais dinâmicos e poderosos:Variáveis ​​do sistemaeVariáveis ​​de entrada personalizadas

Variáveis ​​do sistemasão substituídos automaticamente pelo valor correspondente quando o prompt é usado. Eles são úteis para inserir informações dinâmicas, como a data atual ou os detalhes do usuário.

  • Conteúdo da área de transferência: Usar{{CLIPBOARD}}Para inserir o conteúdo da sua área de transferência.
  • Data e hora
    • {{CURRENT_DATE}}: Data atual
    • {{CURRENT_DATETIME}}: Data e hora atuais
    • {{CURRENT_TIME}}: Horário atual
    • {{CURRENT_TIMEZONE}}: Fuso horário atual
    • {{CURRENT_WEEKDAY}}: Dia atual da semana
  • Informações do usuário
    • {{USER_NAME}}: Nome do usuário atual
    • {{USER_LANGUAGE}}: Linguagem selecionada do usuário
    • {{USER_LOCATION}}: Localização do usuário (requer https e configurações> interface alternam)

Variáveis ​​de entrada personalizadasTransforme seus prompts em modelos interativos. Quando você usa um prompt que contém essas variáveis, uma janela modal aparecerá automaticamente, permitindo que você preencha seus valores. Isso é extremamente poderoso para criar instruções complexas e reutilizáveis ​​que funcionam como formas. Veja as diretrizes abaixo para uma explicação completa.

Ao alavancar variáveis ​​de entrada personalizadas, você pode ir além do texto estático e criar ferramentas interativas diretamente na interface de bate -papo. Esse recurso foi projetado para ser infalível, permitindo que até usuários não técnicos executem instruções complexas e de várias etapas com facilidade. Em vez de editar manualmente um grande bloco de texto, os usuários recebem um formulário limpo e estruturado. Isso não apenas simplifica o fluxo de trabalho, mas também reduz erros, orientando o usuário a fornecer exatamente as informações corretas no formato certo. Ele desbloqueia um novo nível de design de prompt interativo, tornando o uso sofisticado de IA acessível a todos.

Diretrizes de uso variável

  • Coloque todas as variáveis ​​com aparelhos duplos encaracolados:{{variable}}
  • O{{USER_LOCATION}}A variável do sistema requer:
    • Uma conexão HTTPS segura
    • Permitindo o recurso emSettingsInterface
  • O{{CLIPBOARD}}A variável do sistema requer permissão de acesso à área de transferência do seu dispositivo

Usando variáveis ​​de entrada personalizadas

Como funciona

  1. Crie um promptcom uma ou mais variáveis ​​personalizadas usando a sintaxe abaixo.
  2. Use o comando Slash do promptNa entrada de bate -papo.
  3. UmJanela pop -up "variáveis ​​de entrada"aparecerá com um campo de formulário para cada variável que você definiu.
  4. Preencha o formulárioe cliqueSave
  5. As variáveis ​​em seu prompt serão substituídas pela sua entrada e o prompt final será enviado ao modelo.

Sintaxe

Existem duas maneiras de definir uma variável personalizada:

  1. Entrada simples{{variable_name}}

    • Isso cria uma linha única e únicatextDigite o campo de entrada na janela pop -up.

  2. Entrada digitada{{variable_name | [type][:property="value"]}}

    • Isso permite especificar o tipo de campo de entrada (por exemplo, um suspensão, um selecionador de data) e configurar suas propriedades.

Exemplo de uso de casos

1. Summarizador de artigo simples

Crie um prompt reutilizável para resumir qualquer artigo.

  • Comando: /summarize_article
  • Conteúdo imediato: 
    Resuma o seguinte artigo em três principais pontos de bala. Seja conciso e claro.

    Artigo:
    {{article_text | textarea:placeholder="Paste the full text of the article here..."}}
    Quando você digita/summarize_article, um modal aparecerá com uma grande área de texto, levando você a colar o texto do artigo.

2. Gerador de relatório de bug avançado

Esse prompt atua como um formulário estruturado para garantir que todos os detalhes necessários para um relatório de bug sejam capturados.

  • Comando: /bug_report
  • Conteúdo imediato: 
    Gere um relatório de bug com os seguintes detalhes:

    ** Resumo: ** {{resumo | Texto: Parholder = "Um breve resumo da edição"}}
    ** Prioridade: ** {{prioridade | Selecione: opções = ["alta", "médio", "baixo"]: default = "médio"}}
    ** Etapas para reproduzir: **
    {{steps | textarea:placeholder="1. Go to...\n2. Click on...\n3. See error..."}}

    Formate -o em um documento de relatório de bug claro e completo.
    Quando usado, esse prompt gera um formulário com uma entrada de texto, um menu suspenso e duas áreas de texto.

3. Pós -gerador de mídia social

Este prompt gera conteúdo personalizado para diferentes plataformas de mídia social.

  • Comando: /social_post
  • Conteúdo imediato: 
    Gere uma postagem de mídia social para a plataforma {{plataform}}.

    ** Tópico: ** {{tópico | Texto: espaço reservado = "por exemplo, novo recurso de lançamento"}}
    ** Mensagem da chave: ** {{key_message | Textarea: placeholder = "Quais são os pontos essenciais para se comunicar?"}}
    ** Tom da voz: ** {{Tone | Selecione: Opções = ["Profissional", "Casual", "Humanored", "Inspirational"]: default = "Professional"}}
    ** Call à ação: ** {{cta | Texto: espaço reservado = "por exemplo, 'Saiba mais', 'Inscreva -se hoje'"}}

    Inclua hashtags relevantes.

4. Minutos de encontro assistente

Gere rapidamente atas estruturadas da reunião preenchendo um formulário simples.

  • Comando: /meeting_minutes
  • Conteúdo imediato: 
    # Atas da reunião

    ** Data: ** {{Meeting_date | data}}
    ** Hora: ** {{Meeting_time | tempo}}
    ** Participantes: ** {{participantes | Texto: espaço reservado = "Lista de nomes separada por vírgula"}}

    ## Agenda / Pontos de discussão -chave
    {{agenda_items | textarea:placeholder="Paste the agenda or list the key topics discussed."}}

    ## itens de ação
    {{action_items | textarea:placeholder="List each action item, who it is assigned to, and the deadline."}}

    Formate as informações acima em um resumo da reunião limpa e profissional.

Controle de acesso e permissões

O gerenciamento rápido é controlado pelas seguintes configurações de permissão:

  • Solicita acesso: Os usuários precisam doUSER_PERMISSIONS_WORKSPACE_PROMPTS_ACCESSpermissão para criar e gerenciar instruções.
  • Para obter informações detalhadas sobre a configuração de permissões, consulte oDocumentação de permissões

Práticas recomendadas

  • Use títulos claros e descritivos para seus avisos
  • Crie comandos de barra intuitivos que refletem o propósito do prompt
  • Para variáveis ​​personalizadas, use nomes claros (por exemplo,{{your_name}}em vez de{{var1}}) e descritivoplaceholdertexto para facilitar o entendimento dos modelos.
  • Documente quaisquer requisitos específicos ou insumos esperados na descrição rápida
  • Promotos de teste com diferentes combinações variáveis ​​para garantir que funcionem como pretendido
  • Considere os níveis de acesso cuidadosamente ao compartilhar instruções com outros usuários - compartilhamento público significa que ele aparecerá automaticamente para todos os usuários quando eles acertarem/Em um bate -papo, você deseja evitar criar muitos.
