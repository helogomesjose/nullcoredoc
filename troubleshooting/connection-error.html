🚧 Problemas de conectividade do servidor

Estamos aqui para ajudá -lo a configurar tudo bem. Abaixo, você encontrará instruções passo a passo adaptadas para diferentes cenários para resolver problemas comuns de conexão com Ollama e servidores externos, como abraçar o rosto.

🌟 Conexão com o servidor Ollama

🚀 Acessando o Ollama da Open Webui

Lutando para se conectar a Ollama a partir do Open Webui? Pode ser porque Ollama não está ouvindo uma interface de rede que permite conexões externas. Vamos resolver isso:

  1. Configure Ollama para ouvir de maneira ampla🎧: DefinirOLLAMA_HOSTpara0.0.0.0Para fazer o Ollama ouvir em todas as interfaces de rede.

  2. Atualizar variáveis ​​de ambiente: Certifique -se de que oOLLAMA_HOSTestá definido com precisão em seu ambiente de implantação.

  3. Reinicie Ollama🔄: É necessário reinício para que as alterações entrem em vigor.

💡 Após a configuração, verifique se Ollama está acessível visitando a interface Webui.

Para obter instruções mais detalhadas sobre como configurar o ollama, consulte oDocumentação oficial de Ollama

Error de conexão do docker

Se você está vendo um erro de conexão ao tentar acessar o Ollama, pode ser porque o contêiner do Webui Docker não pode conversar com o servidor Ollama em execução no seu host. Vamos consertar isso:

  1. Ajuste as configurações de rede🛠️: Use o--network=hostSinalize no seu comando docker. Isso vincula seu contêiner diretamente à rede do seu host.

  2. Altere a porta: Lembre -se de que a porta interna muda de 3000 para 8080.

Exemplo de comando docker

Docker Run -d--network = host -V Open-Webui:/App/backend/data -e ollama_base_url = http: //127.0.0.1: 11434--name Open-Webui ----Restart sempre ghcr.io/open-webui/open-webui ----restart sempre ghcr.io/open-webui/open-webui ----restart sempre

🔗 Depois de executar o exposto, seu webui deve estar disponível emhttp://localhost:8080

🔒 Problema de conexão SSL com o rosto de abraçar

Encontrou um erro SSL? Pode ser um problema com o Hugging Face Server. Aqui está o que fazer:

  1. Verifique o status do servidor de face de abraço: Verifique se há uma interrupção ou questão conhecida.

  2. Interruptor de terminal: Se o rosto de abraçar está baixo, trocam o ponto de extremidade no comando do Docker.

Exemplo de comando docker para problemas conectados

docker run -d -p 3000:8080 -e HF_ENDPOINT=https://hf-mirror.com/ --add-host=host.docker.internal:host-gateway -v open-webui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:main

🍏 Podman no macOS

Correndo no macOS com podman? Veja como garantir a conectividade:

  1. Ativar loopback do host: Usar--network slirp4netns:allow_host_loopback=trueem seu comando.

  2. Definir ollama_base_url: Garantir que aponta parahttp://host.containers.internal:11434

Exemplo de comando podman

podman run -d - -network slirp4Netns: allow_host_loopback = true -p 3000: 8080 -e ollama_base_url = http: //host.containers.internal: 11434 -V webui:/app/backend/ - -nam ghcr.io/open-webui/open-webui:Main
